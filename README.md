# SM4 分组密码 C 语言实现

本项目使用 C 语言实现了 SM4 分组密码算法，提供了：

- SM4 核心轮函数与密钥扩展
- 单块加解密
- ECB 模式加解密
- PKCS#7 填充
- 命令行十六进制输入与输出

## 算法参数

- 分组长度：128 bit（16 字节）
- 密钥长度：128 bit（16 字节）
- 轮数：32 轮
- 当前实现模式：ECB
- 填充方式：PKCS#7

## 输入输出格式

- 密钥输入：16 进制字符串，固定 32 个十六进制字符
- 明文输入：16 进制字符串，长度可变
- 密文输出：16 进制字符串
- 解密输入：16 进制字符串，长度必须是 16 字节分组的整数倍
- 解密输出：16 进制字符串

说明：

- 命令行接口当前按“字节流的十六进制表示”处理输入和输出，不直接接受十进制数据。
- 如果需要处理文本，可先将文本转成十六进制，再传给程序。

## 使用方法

### 1. 加密

```bash
./sm4 enc 0123456789abcdeffedcba9876543210 0123456789abcdeffedcba9876543210
```

说明：

- 第 1 个参数 `enc` 表示加密
- 第 2 个参数是 128 bit 密钥的 16 进制表示
- 第 3 个参数是明文的 16 进制表示

注意：

- 本项目的命令行加密接口使用 ECB + PKCS#7，因此即使输入正好是 16 字节，也会补一整块填充。
- 所以上述命令输出的是“带填充后的完整密文”，不是单纯一个裸分组的标准向量结果。

### 2. 解密

```bash
./sm4 dec 0123456789abcdeffedcba9876543210 <ciphertext_hex>
```

说明：

- 第 1 个参数 `dec` 表示解密
- 第 2 个参数是密钥的 16 进制表示
- 第 3 个参数是密文的 16 进制表示

程序会在解密后自动去掉 PKCS#7 填充，并输出原始明文的十六进制字符串。

### 3. 单分组加密（无填充）

```bash
./sm4 enc-block 0123456789abcdeffedcba9876543210 0123456789abcdeffedcba9876543210
```

说明：

- `enc-block` 只处理单个 128 bit 分组
- 输入必须恰好是 32 个十六进制字符
- 不执行 PKCS#7 填充

### 4. 单分组解密（无填充）

```bash
./sm4 dec-block 0123456789abcdeffedcba9876543210 681edf34d206965e86b3e94f536e4246
```

说明：

- `dec-block` 只处理单个 128 bit 分组
- 输入必须恰好是 32 个十六进制字符
- 不执行去填充操作

## 标准测试向量

SM4 常见标准单块测试向量如下：

- 密钥：`0123456789abcdeffedcba9876543210`
- 明文：`0123456789abcdeffedcba9876543210`
- 单块密文：`681edf34d206965e86b3e94f536e4246`

使用本项目可以直接验证：

```bash
./sm4 enc-block 0123456789abcdeffedcba9876543210 0123456789abcdeffedcba9876543210
```

预期输出：

```text
681edf34d206965e86b3e94f536e4246
```
